# cuda-bindings

Raw FFI bindings to the CUDA driver API, generated by [bindgen](https://crates.io/crates/bindgen) from `wrapper.h` (which includes `cuda.h`).

## What this crate provides

- All types, constants, and function declarations from `cuda.h` (e.g., `cuInit`, `cuLaunchKernel`, `CUdeviceptr`, `CUstream`)

## Build requirements

- CUDA Toolkit installed (headers in `<toolkit>/include/cuda.h` or `<toolkit>/targets/<dir>/include/cuda.h`)
- `clang` available for bindgen (typically bundled with LLVM)
- On Windows, the `x86_64-pc-windows-msvc` Rust target and the CUDA driver import library at `<toolkit>\lib\x64\cuda.lib`

The build script locates the toolkit with shared workspace discovery. Root search order is `CUDA_TOOLKIT_PATH`, `CUDA_HOME`, `CUDA_PATH`, `CUDA_PATH_V*`, Windows defaults such as `C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.3`, then Linux defaults `/usr/local/cuda` and `/opt/cuda`.

Headers are discovered in `<toolkit>/include/cuda.h` or, for CUDA redistributable layouts without a top-level `include/`, in the target-specific `targets/<dir>/include/cuda.h` directory. When `cuda.h` is not found, the build fails with one message listing the probed paths instead of a bindgen panic.

For Linux targets it searches `<toolkit>/lib64`, `<toolkit>/lib64/stubs`, and redistributable `targets/<dir>/lib` layouts. For Windows MSVC targets it searches `<toolkit>\lib\x64` and links `cuda`.

## Environment variables

| Variable            | Purpose                                       | Notes                                      |
|---------------------|-----------------------------------------------|--------------------------------------------|
| `CUDA_TOOLKIT_PATH` | Preferred CUDA Toolkit root                   | Wins over `CUDA_HOME` and `CUDA_PATH`      |
| `CUDA_HOME`         | CUDA Toolkit root                              | Checked after `CUDA_TOOLKIT_PATH`          |
| `CUDA_PATH`         | CUDA Toolkit root, common on Windows           | Checked before versioned `CUDA_PATH_V*`    |
| `CUDA_PATH_V13_3`   | Versioned CUDA Toolkit root from the installer | Rebuild trigger emitted explicitly         |

## Usage

This crate is not intended for direct use. Prefer the safe wrappers in `cuda-core` or the async layer in `cuda-async`.

```rust
// Low-level access (if you need it):
use cuda_bindings::{cuInit, cuDeviceGet, CUdevice};
```
